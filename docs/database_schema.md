# Database Schema

## Project Name

AtmoSync - Micro-Climate Arbitrage Analytics

---

## Overview

The AtmoSync project uses IoT-based sensor data to monitor the environmental conditions of perishable commodities during transportation. The database schema has been designed to support real-time data ingestion, spoilage prediction, market analysis, and business intelligence reporting.

---

## Datasets Used

The project consists of six datasets:

1. container_master.csv
2. shipment_master.csv
3. container_sensor_data.csv
4. commodity_prices.csv
5. market_locations.csv
6. spoilage_rules.csv

---

# Dataset Schema

## 1. container_master.csv

### Description

Stores information about refrigerated containers used for transportation.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| container_id | VARCHAR | Unique Container ID (Primary Key) |
| container_type | VARCHAR | Type of Container |
| capacity_kg | INTEGER | Maximum Capacity |
| manufacturer | VARCHAR | Manufacturer Name |
| installation_date | DATE | Installation Date |
| status | VARCHAR | Active / Inactive |
| battery_percentage | INTEGER | Remaining Battery Percentage |

### Primary Key

- container_id

---

## 2. shipment_master.csv

### Description

Stores shipment-related information.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| shipment_id | VARCHAR | Unique Shipment ID |
| container_id | VARCHAR | Associated Container ID |
| commodity_name | VARCHAR | Commodity Name |
| source_location | VARCHAR | Shipment Origin |
| destination_location | VARCHAR | Shipment Destination |
| shipment_date | DATE | Shipment Date |
| expected_delivery | DATE | Expected Delivery Date |
| shipment_status | VARCHAR | In Transit / Delivered |

### Primary Key

- shipment_id

### Foreign Key

- container_id

References:

- container_master(container_id)

---

## 3. container_sensor_data.csv

### Description

Stores real-time IoT sensor readings generated by containers.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| sensor_id | VARCHAR | Unique Sensor ID |
| shipment_id | VARCHAR | Shipment Identifier |
| container_id | VARCHAR | Container Identifier |
| temperature_c | FLOAT | Temperature Reading |
| humidity_percentage | FLOAT | Humidity Reading |
| latitude | FLOAT | GPS Latitude |
| longitude | FLOAT | GPS Longitude |
| battery_percentage | INTEGER | Battery Percentage |
| timestamp | DATETIME | Sensor Reading Time |

### Primary Key

- sensor_id

### Foreign Keys

- shipment_id
- container_id

References:

- shipment_master(shipment_id)
- container_master(container_id)

---

## 4. commodity_prices.csv

### Description

Stores commodity prices across different markets.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| commodity_id | VARCHAR | Commodity Identifier |
| commodity_name | VARCHAR | Commodity Name |
| market_id | VARCHAR | Market Identifier |
| price_per_kg | DECIMAL | Current Market Price |
| last_updated | DATE | Last Updated Date |

### Primary Key

- commodity_id

### Foreign Key

- market_id

References:

- market_locations(market_id)

---

## 5. market_locations.csv

### Description

Stores market-related information.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| market_id | VARCHAR | Unique Market ID |
| market_name | VARCHAR | Market Name |
| city | VARCHAR | City Name |
| state | VARCHAR | State Name |
| latitude | FLOAT | Latitude Coordinate |
| longitude | FLOAT | Longitude Coordinate |

### Primary Key

- market_id

---

## 6. spoilage_rules.csv

### Description

Stores business rules for spoilage prediction.

| Column Name | Data Type | Description |
|------------|-----------|------------|
| rule_id | VARCHAR | Unique Rule ID |
| commodity_name | VARCHAR | Commodity Name |
| minimum_temperature | FLOAT | Minimum Allowed Temperature |
| maximum_temperature | FLOAT | Maximum Allowed Temperature |
| minimum_humidity | FLOAT | Minimum Humidity |
| maximum_humidity | FLOAT | Maximum Humidity |
| spoilage_percentage | FLOAT | Predicted Spoilage Percentage |

### Primary Key

- rule_id

# Business Purpose

The database schema is designed to support:

- Real-time shipment monitoring.
- Temperature and humidity analysis.
- Spoilage prediction.
- Commodity price analysis.
- Market intelligence.
- Business alert generation.
- Executive dashboard reporting.
